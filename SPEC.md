# Truchet Tile Studio — ゲームモード仕様書

## 確定仕様

| 項目 | 決定内容 |
|------|---------|
| やめるボタン | なし（デザインタブで戻る） |
| タイマー表示 | `mm:ss.ms`、スタートボタンと差し替え |
| 一時停止ボタン | シャッフルボタンと差し替え |
| オブジェクト | カーナビ三角、道幅連動、接線方向に回転 |
| 速度カーブ | `speed = base * e^(a * elapsed)`、aは小さめ |
| ハイライト | 白オーバーレイ、前方グラデーション、速度で短く薄く |
| 終了条件A | デッドエンド到達 |
| 終了条件B | タイル遷移時に遷移先が回転中 |
| ブラウザバック | 強制終了・記録なし |
| デザインタブ押下 | 一時停止メニュー（再開／最初から／やめる） |
| ループ | 一周でタイマーグレーアウト停止、速度は継続上昇 |
| ランキング | localStorage上位3位、キャンバス中央オーバーレイ、背景半透明グレー |
| スコア保存 | localStorage |

## データ設計

### 道の表現
```js
// タイル内の道はローカル座標で保持（回転に強い）
road = { edgeA: {edge, t}, edgeB: {edge, t}, warp? }
// 回転後: edgeA.edge = (edge + rot) % n
```

### グラフ構造
- ノード = (tileId, segmentIdx, side)
- タイル境界接続 = 辺番号 + 隣接タイル座標テーブル
- タイル境界でのt値反転：出口t → 入口 `1-t`

### 先読みアルゴリズム
1. 現在位置から経路を前向きに辿る
2. 既訪問ノード出現 → ループ確定
3. グリッド外到達 → デッドエンド確定
4. タイル回転のたびに再計算

## UI状態遷移

```
待機中
  └─[スタート押下]→ プレイ中
                       ├─[デッドエンド到達]→ 終了（記録あり）
                       ├─[条件B発動]→ 終了（記録あり）
                       ├─[ブラウザバック]→ 終了（記録なし）
                       ├─[デザインタブ押下]→ 一時停止
                       └─[一時停止ボタン]→ 一時停止
                                              ├─[再開]→ プレイ中
                                              ├─[最初から]→ 待機中
                                              └─[やめる]→ 待機中
```

## 未調整パラメータ（実装後に調整）
- 初速の値
- 速度上昇係数 a
- ハイライトが消える速度閾値
- ランクインアニメーション詳細
